##Complexity Reflectrion


a. Search element in the first position: 
      This is the Best Case scenario.
      The algorithm performs 1 operation (one comparison) and returns the result immediately.
-------------------------------------------------------------------------------------------------
b. Search element at the end: 
      This is the Worst Case (or near-worst case).
      The algorithm must iterate through every element, performing "n" operations.
-------------------------------------------------------------------------------------------------
c. Search for the element in the middle:
     On average, the algorithm will perform n/2 operations.
--------------------------------------------------------------------------------------------------
d. Big O notation for the algorithm: The Big O notation is O(n), which represents Linear Time Complexity.
    This is because the number of operations grows proportionally with the number of elements in the array.
